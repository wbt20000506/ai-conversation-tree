# ChatGPT Tree Panel

一个运行在 `chatgpt.com` 的 Chrome 扩展，用来把当前对话整理成可搜索、可折叠、可重排的树状面板。

## 安装方法

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择项目目录 `chatgpt-tree-plugin`
5. 打开 `https://chatgpt.com/` 并进入任意对话

## 核心功能

- 在 ChatGPT 页面右侧显示对话树
- 自动分析问题之间的父子关系
- 未配置 API 或 AI 失败时自动回退到本地硬算法
- 支持搜索问题，并用 `上一个` / `下一个` 跳转
- 支持拖拽节点，把一个问题拖到另一个问题下面
- 支持将当前问题设为上级问题、下级问题或根问题
- 支持删除当前问题，并在当前对话里忽略它
- 支持 `上一状态`，可撤销最近一次结构操作
- 支持 `重新排序`，按当前对话重新分析整棵树
- 支持保存当前树结构，作为后续 AI 分析参考
- 支持导出为 `Markdown`、`PNG`、`SVG`、`JPG`
- 支持记住同一对话上次的树状态、折叠状态和忽略状态

## 导出说明

- `Markdown`
  - `完整对话`：导出当前树里保留的问题和对应回答
  - `仅问题`：只导出问题树
- `PNG / SVG / JPG`
  - 只导出问题树
- 导出文件名会优先使用 ChatGPT 当前会话标题，并附带导出时间

## AI 接口支持

- OpenAI
- Gemini
- Claude
- DeepSeek
- MiMo
- 自定义 OpenAI 兼容端点

说明：

- 模型名直接手输，不强制下拉选择
- 配置会自动保存在 `chrome.storage.local`
- 若上次验证 API 可用，刷新后会保留可用状态
- 点 `重新排序` 时会优先使用 AI；AI 不可用时自动回退硬算法
- 点弹窗里的 `清空` 后，会断开当前 API 配置，后续重新排序使用硬算法

## 权限说明

扩展只会注入到 `chatgpt.com`。

`manifest.json` 中保留了 `https://*/*` 和 `http://*/*`，是为了支持自定义 OpenAI 兼容端点。若你不需要这个能力，可以自行删除这两个通配权限。


## 设置方法

1. 点击浏览器工具栏中的扩展图标
2. 选择 `API 类型`
3. 输入模型名，可留空
4. 输入对应的 `API Key`
5. 如果使用自定义端点，再填写完整接口地址
6. 点击 `保存`

说明：

- 弹窗中的配置会自动记住
- `保存` 会额外验证当前接口是否可用
- 未配置 API Key 时，会自动使用硬算法

## 使用说明

- 面板默认显示在页面右侧
- 点击问题标题或圆点可跳转到对应提问
- 点击方形折叠按钮可展开或收起子树
- 面板会跟随当前页面滚动，高亮当前问题
- `删除问题` 会让当前问题从树、搜索和导出中消失
- 刷新网页后，被删除的问题仍会保持忽略
- `重新排序` 会重新拉回所有问题，再按当前对话重新分析
- `上一状态` 可回到最近一次操作前的树状态

## 项目结构

- `manifest.json`：MV3 配置
- `content.js`：主面板逻辑、扫描、树渲染、导出、交互
- `content.css`：面板与树样式
- `content-core.js`：AI 分析 prompt 与结果解析
- `hard-algorithm.js`：本地硬算法提取与关系推断
- `background.js`：后台请求代理
- `popup.html` / `popup.js` / `popup.css`：AI 接口配置弹窗

## 已知限制

- 对话树基于当前页面 DOM 构建，不调用 ChatGPT 私有接口
- ChatGPT 页面没有稳定暴露每条消息的真实时间，因此树中不显示提问时间
- 页面结构若发生较大改版，扫描选择器可能需要调整
- 图片导出基于当前树视图，受当前展开状态影响
